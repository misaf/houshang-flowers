# Frontend Guide

Use this file as the entry point for frontend work. Read the always-read files, then only the focused files that match the task.

## Always Read For Frontend Work

- `docs/frontend/core.md` for principles, AI workflow, and Next.js App Router rules.
- `docs/frontend/design-system.md` for shadcn/ui, visual style, theme, and motion rules.
- `docs/frontend/quality.md` for mobile, accessibility, performance, data integrity, i18n, images, forms, and "never" rules.

## Read When Relevant

- `docs/frontend/navigation.md` for header, desktop navigation, mobile navigation, and navigation behavior.
- `docs/frontend/homepage.md` for homepage, hero, category sections, blog section, and storefront scroll behavior.
- `docs/frontend/products.md` for product cards and product detail pages.
- `docs/frontend/blog.md` for the blog listing, blog post detail, and homepage blog section.
- `docs/frontend/faq.md` for the FAQ page (categories, search, accordion) and FAQ JSON-LD.
- `docs/frontend/cart-and-checkout.md` for cart, favorites, account panel, and the demo checkout flow.
- `docs/frontend/about-us.md` for the about-us page, brand story, values, process, and trust-building content.
- `docs/frontend/contact.md` for the contact-us page, contact information, and inquiry form behavior.
