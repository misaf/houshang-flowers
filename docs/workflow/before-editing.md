# Workflow Before Editing

Follow this workflow for every code change in this repository.

## Before Editing

Before writing code:

1. Inspect the surrounding codebase to understand the existing architecture.
2. Review relevant components, hooks, API clients, resource mappers, contexts, utilities, and translations.
3. Determine the appropriate location for the change:
   - Server Component
   - Client Component
   - Shared UI component
   - API layer
   - Context
   - Hook
   - Utility
4. Reuse existing patterns before introducing new abstractions.
5. For non-trivial work, create a brief implementation plan before making changes.
6. Keep the scope focused on the requested task.
